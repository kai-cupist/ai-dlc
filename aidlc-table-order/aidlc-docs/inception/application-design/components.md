# 컴포넌트 정의

## 백엔드 컴포넌트 (Python + FastAPI)

### 아키텍처 패턴
**계층형**: Router → Service → Repository (+ TDD)

```
app/                              cores/
+------------------+              +------------------+
|    Routers       |  API 처리    |    security/     |  JWT, Rate Limiter
+------------------+              +------------------+
        |                         |    config.py     |  앱 설정
+------------------+              +------------------+
|    Services      |  비즈니스    |    logging.py    |  구조화된 로깅
+------------------+              +------------------+
        |                         |    db/           |
+------------------+              |  +-------------+ |
|  Repositories    |  데이터 접근  |  |   models/   | |  SQLAlchemy 모델
+------------------+              |  +-------------+ |
        |                         |  | session.py  | |  DB 엔진/세션
+------------------+              |  +-------------+ |
|    DTO           |  요청/응답    +------------------+
+------------------+
```

---

### C1: Auth (인증)
| 항목 | 내용 |
|------|------|
| **목적** | 관리자/테이블 인증 및 세션 관리 |
| **책임** | JWT 토큰 발급/검증, 비밀번호 해싱, 로그인 시도 제한, 세션 관리 |
| **인터페이스** | 관리자 로그인, 테이블 초기 설정, 테이블 자동 로그인, 토큰 갱신 |

### C2: Store (매장)
| 항목 | 내용 |
|------|------|
| **목적** | 매장 정보 관리 |
| **책임** | 매장 데이터 조회, 매장별 설정 관리 |
| **인터페이스** | 매장 정보 조회 |

### C3: Table (테이블)
| 항목 | 내용 |
|------|------|
| **목적** | 테이블 및 세션 라이프사이클 관리 |
| **책임** | 테이블 CRUD, 세션 시작/종료, 이용 완료 처리, 과거 내역 이동 |
| **인터페이스** | 테이블 목록 조회, 세션 종료(이용 완료), 과거 주문 내역 조회 |

### C4: Menu (메뉴)
| 항목 | 내용 |
|------|------|
| **목적** | 메뉴, 카테고리, 옵션 그룹 관리 |
| **책임** | 메뉴 CRUD, 카테고리 관리, 옵션 그룹 CRUD, 메뉴-옵션 연결, 노출 순서 관리 |
| **인터페이스** | 메뉴 조회(카테고리별), 메뉴 CRUD, 옵션 그룹 CRUD, 메뉴-옵션 연결 |

### C5: Order (주문)
| 항목 | 내용 |
|------|------|
| **목적** | 주문 생성, 상태 관리, 이력 관리 |
| **책임** | 주문 생성(회차 관리), 상태 변경, 주문 삭제, 주문 목록 조회, 폴링 데이터 제공, 주문 이력 아카이빙 |
| **인터페이스** | 주문 생성, 주문 목록 조회, 상태 변경, 주문 삭제, 폴링 엔드포인트, 과거 이력 조회 |

### C6: Recommendation (추천)
| 항목 | 내용 |
|------|------|
| **목적** | 주문 히스토리 기반 메뉴 추천 |
| **책임** | 장바구니 메뉴 기반 함께 주문된 메뉴 분석, 추천 메뉴 없을 시 랜덤 추천 |
| **인터페이스** | 추천 메뉴 조회 (장바구니 메뉴 ID 목록 입력) |

## 프론트엔드 컴포넌트 (모노레포)

### 상태 관리 전략
- **서버 상태**: TanStack Query (React Query) - API 데이터 캐싱, 폴링
- **로컬 상태**: React Context API + useReducer - 장바구니, UI 상태

### 모노레포 구조

```
+---------------------------------------------+
|                  monorepo                    |
|  +------------------+  +------------------+ |
|  |  apps/customer   |  |  apps/admin      | |
|  |  (모바일 최적화)  |  |  (데스크톱 최적화) | |
|  +--------+---------+  +--------+---------+ |
|           |                      |           |
|  +--------+----------------------+---------+ |
|  |          packages/shared                | |
|  |  (API 클라이언트, 타입, 공통 훅)        | |
|  +-----------------------------------------+ |
+---------------------------------------------+
```

### F1: packages/shared (공유 패키지)
| 항목 | 내용 |
|------|------|
| **목적** | 고객/관리자 앱 간 공유 코드 |
| **책임** | API 클라이언트, TypeScript 타입 정의, 공통 유틸리티, 공통 훅 |

### F2: apps/customer (고객용 앱)
| 항목 | 내용 |
|------|------|
| **목적** | 고객 주문 인터페이스 |
| **책임** | 메뉴 탐색, 장바구니, 주문, 대기 화면, 주문 내역 |
| **최적화** | 모바일 웹, 터치 친화적, 카테고리 사이드바 |

### F3: apps/admin (관리자용 앱)
| 항목 | 내용 |
|------|------|
| **목적** | 매장 관리 인터페이스 |
| **책임** | 로그인, 실시간 대시보드, 테이블 관리, 메뉴/옵션 관리 |
| **최적화** | 데스크톱 웹, 그리드 레이아웃, 폴링 기반 업데이트 |
