| **백엔드** | Python + FastAPI | 비동기 지원, REST API |
| **프론트엔드 (고객용)** | React + Vite (SPA) | 모바일 웹 최적화 |

| **이미지 저장소** | AWS S3 | 메뉴 이미지 업로드/관리 |

| **UI 라이브러리** | Tailwind CSS + shadcn/ui | 다크 프리미엄 테마 |

## 기능 요구사항

### FR-01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자 1회 초기 설정 (매장ID, 테이블번호, 비밀번호)
- 설정 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지

### FR-02: 메뉴 조회 및 탐색
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세: 메뉴명, 가격, 설명, 이미지
- **인기 메뉴 뱃지**: 이미지 영역 오른쪽 상단 표시
- **메뉴 옵션 선택**: 옵션 그룹별 UI (맵기, 사이즈, 토핑 등)
  - 필수/선택 옵션 구분, 추가 가격 표시
  - 옵션 없는 메뉴는 상세 화면 없이 바로 장바구니 추가
- **카테고리 사이드바**: 왼쪽 사이드바에 카테고리 목록
  - 스크롤 시 현재 카테고리 자동 하이라이트
  - 카테고리 클릭 시 해당 영역으로 이동
- **카테고리별 영역 구분**: 소제목 + 구분선
- 카드 레이아웃, 터치 친화적 (최소 44x44px)

### FR-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- **옵션 정보 포함**: 같은 메뉴+다른 옵션 = 별도 항목
- 총 금액 실시간 계산 (옵션 추가 가격 포함)
- 장바구니 비우기
- 클라이언트 로컬 저장 (새로고침 유지)
- 서버 전송은 주문 확정 시에만

### FR-04: 함께 주문한 메뉴 추천
- **표시 시점**: "주문하기" 클릭 시 바텀시트로 등장
- **추천 로직**: 주문 히스토리 기반 - 현재 장바구니 메뉴와 함께 주문된 적 있는 메뉴 중 장바구니에 없는 것 추천
- 추천 메뉴 선택 시 장바구니 추가
- "추천 건너뛰기" / "바로 주문하기" 버튼
- 추천 메뉴 없으면 바텀시트 없이 바로 주문 확정

### FR-05: 주문 생성
- 주문 내역 최종 확인
- **계산 안내 멘트**: "계산은 매장 나가기 할 때 프론트에서 해주세요"
- 주문 확정 버튼
- 주문 성공: 주문번호 표시, 장바구니 비우기, 메뉴 화면 리다이렉트
- 주문 실패: 에러 메시지 + 장바구니 유지
- **주문 정보**: 매장ID, 테이블ID, 메뉴 목록 (메뉴명, 수량, 단가, 선택 옵션, 옵션 추가 가격), 총 금액, 세션ID, 주문 회차

### FR-06: 주문 대기 및 진행 상황
- **진행 단계**: 주문 접수 → 준비 중 → 서빙 대기
- **모래시계 애니메이션**: "준비 중" 단계에서 예상 남은 시간을 모래시계 + % 진행률로 시각화
- 추가 주문 시 예상 시간 증가 반영
- **영수증 확인**: 대기 화면에서 전체 주문 영수증 조회 (회차별 항목, 옵션, 금액, 합계)
- **추가 주문 구분**: "n회차 추가 주문" 뱃지

### FR-07: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별: 주문번호, 시각, 메뉴+수량 (옵션 포함), 금액, 상태, **회차 뱃지**
- **맨 하단 total 주문 금액 표시**
- 현재 테이블 세션 주문만 표시
- 페이지네이션 또는 무한 스크롤

### FR-08: 관리자 매장 인증
- 매장ID + 사용자명 + 비밀번호
- JWT 토큰 기반, 16시간 세션
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

### FR-09: 실시간 주문 모니터링
- **그리드/대시보드 레이아웃**: 테이블별 카드 형태
- 각 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 → 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조 (색상, 애니메이션)
- **REST API 폴링 (2초 간격)** 기반 업데이트
- 테이블별 필터링

### FR-10: 테이블 관리
- **테이블 초기 설정**: 번호, 비밀번호, 16시간 세션
- **주문 삭제**: 확인 팝업 → 삭제 → 총 주문액 재계산
- **테이블 세션 종료 (이용 완료)**: 확인 팝업 → 주문 내역 과거 이력 이동 → 리셋
- **과거 주문 내역**: 시간 역순, 날짜 필터링
- 성공/실패 피드백

### FR-11: 메뉴 관리
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지 URL)
- 메뉴 노출 순서 조정
- **옵션 그룹 관리**:
  - 옵션 그룹 CRUD (예: "맵기 단계", "사이즈", "토핑")
  - 옵션 항목별 추가 가격 설정
  - 필수/선택 옵션 그룹 구분
  - 메뉴-옵션 그룹 연결 (N:M)
  - 옵션 그룹 재사용 (여러 메뉴에 동일 그룹 적용)
- 필수 필드 검증, 가격 범위 검증

## 비기능 요구사항

### NFR-01: 성능
- 주문 모니터링 폴링 간격: 2초
- 2초 이내 주문 표시 목표
- 대규모 지원: 다중 매장, 매장당 50+ 테이블

### NFR-02: 보안 (Security Baseline 적용)
- SECURITY-01 ~ SECURITY-15 규칙 적용
- 비밀번호 bcrypt 해싱 (SECURITY-12)
- JWT 토큰 서버 측 검증 (SECURITY-08)
- 입력값 검증 및 SQL 인젝션 방지 (SECURITY-05)
- HTTP 보안 헤더 (SECURITY-04)
- 구조화된 로깅 (SECURITY-03)
- Rate limiting (SECURITY-11)
- 에러 핸들링 fail-safe (SECURITY-15)

### NFR-03: 테스트
- 단위 테스트 + API 통합 테스트 + E2E 테스트 (Cypress/Playwright)

### NFR-04: 배포
- Docker 컨테이너화
- 구체적 배포 환경은 추후 결정

### NFR-05: 사용성
- 고객용: 모바일 웹 최적화, 터치 친화적
- 관리자용: 데스크톱 웹 최적화, 그리드 레이아웃

### NFR-06: UI/UX 디자인
- **UI 라이브러리**: Tailwind CSS + shadcn/ui
- **디자인 방향**: 다크 프리미엄 (어두운 배경, 골드/오렌지 포인트, 고급 레스토랑 느낌)
- **색상 테마**:
  - Primary: `#F59E0B` (앰버/골드) - CTA 버튼, 강조
  - Secondary: `#D97706` (다크 앰버) - 보조 강조
  - Background: `#111827` (다크 네이비) - 메인 배경
  - Surface: `#1F2937` (다크 그레이) - 카드, 모달
  - Text Primary: `#F9FAFB` (화이트) - 주요 텍스트
  - Text Secondary: `#9CA3AF` (라이트 그레이) - 보조 텍스트
  - Success: `#10B981` (에메랄드) - 완료, 성공
  - Warning: `#F59E0B` (앰버) - 대기중, 준비중
  - Error: `#EF4444` (레드) - 에러, 삭제
  - Border: `#374151` (그레이) - 구분선, 테두리

## 제외 기능

- 실제 결제 처리, PG사 연동, 영수증 발행/출력, 환불, 포인트/쿠폰
- OAuth, SNS 로그인, 2FA/OTP
- 이미지 리사이징/최적화, CMS, 광고
- 푸시/SMS/이메일/소리/진동 알림
- 주방 전달, 재고 관리
- 데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어
- 배달 플랫폼, POS, 소셜 미디어, 지도/번역 API 연동

## 용어 정의

| 용어 | 정의 |
|------|------|
| **테이블 세션** | 고객 첫 주문 ~ 이용 완료 처리까지의 기간. 종료 후 새 고객의 첫 주문 시 새 세션 시작 |
| **옵션 그룹** | 메뉴에 연결되는 커스터마이징 항목 묶음 (예: 맵기, 사이즈, 토핑). 필수/선택 구분 |
| **주문 회차** | 세션 내 주문 생성 순서. 첫 주문 = 1회차, 추가 주문 = n회차 |
