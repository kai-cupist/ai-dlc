## 고객 페르소나

### P1: 혼밥 고객 - "민지" (25세, 직장인)

| 항목 | 내용 |
|------|------|
| **설명** | 점심시간에 혼자 빠르게 식사하는 직장인 |
| **목표** | 최소한의 인터랙션으로 빠르게 주문하고 식사 완료 |
| **기술 수준** | 높음 - 스마트폰/태블릿 사용에 익숙 |
| **특성** | 메뉴 선택이 빠름, 추천보다 자기 결정 선호, 시간 민감 |
| **이용 패턴** | 단일 메뉴 1~2개, 추가 주문 드묾, 옵션 간단히 선택 |
| **핵심 니즈** | 빠른 주문, 대기 시간 확인, 간결한 UI |
| **불만 포인트** | 불필요한 단계가 많은 주문 과정, 느린 반응 속도 |

### P2: 일반 그룹 고객 - "준호" (32세, 회사원)

| 항목 | 내용 |
|------|------|
| **설명** | 동료 3~4명과 함께 점심/저녁 식사하는 회사원 |
| **목표** | 여러 사람의 메뉴를 한 번에 편리하게 주문 |
| **기술 수준** | 중간 - 기본적인 디지털 기기 사용 가능 |
| **특성** | 메뉴를 꼼꼼히 탐색, 옵션 다양하게 선택, 추가 주문 빈번 |
| **이용 패턴** | 다양한 메뉴 3~5개, 추가 주문 1~2회, 옵션 적극 활용 |
| **핵심 니즈** | 카테고리별 메뉴 탐색, 장바구니 관리, 주문 내역 확인 |
| **불만 포인트** | 메뉴 탐색이 어려운 구조, 주문 현황 파악 불가 |

### P3: 대규모 모임 고객 - "수진" (40세, 모임 총무)

| 항목 | 내용 |
|------|------|
| **설명** | 10명 이상의 모임을 주관하며 주문을 총괄하는 역할 |
| **목표** | 많은 인원의 주문을 정확하게 관리하고 총 금액 파악 |
| **기술 수준** | 중하 - 디지털 기기 기본 사용 가능하나 복잡한 기능은 어려움 |
| **특성** | 여러 번 추가 주문, 영수증 확인 필수, 총 금액 관리 중요 |
| **이용 패턴** | 메뉴 5~10개 이상, 추가 주문 3회 이상, 옵션 다양 |
| **핵심 니즈** | 회차별 주문 구분, 영수증 확인, total 금액 표시, 주문 이력 추적 |
| **불만 포인트** | 추가 주문 구분 불가, 전체 금액 추적 어려움, 복잡한 UI |

## 관리자 페르소나

### P4: 매장 관리자 - "영호" (45세, 식당 사장님)

| 항목 | 내용 |
|------|------|
| **설명** | 중대형 식당을 운영하며 주문 관리와 메뉴 설정을 총괄 |
| **목표** | 실시간 주문 현황 파악, 효율적인 테이블 회전, 메뉴/옵션 관리 |
| **기술 수준** | 중간 - PC 기본 사용 가능, 관리 시스템 학습 의지 있음 |
| **특성** | 바쁜 피크 시간대 빠른 주문 확인 필요, 직관적 UI 선호 |
| **이용 패턴** | 피크 시간 집중 모니터링, 수시 상태 변경, 정기적 메뉴 업데이트 |
| **핵심 니즈** | 실시간 대시보드, 테이블 상태 한눈에 파악, 간편한 메뉴 관리 |
| **불만 포인트** | 주문 누락, 테이블 상태 혼동, 복잡한 메뉴 설정 |
