# 사용자 스토리

> **분류 방식**: 사용자 여정 기반  
> **세분화**: 중간 단위 (주요 사용자 액션당 1개)  
> **인수 기준**: Given-When-Then (BDD)  
> **에러 시나리오**: 포함

## Epic 1: 고객 여정 - 입장 및 세션

### US-01: 테이블 태블릿 자동 로그인
**As a** 고객 (P1, P2, P3)  
**I want** 테이블에 앉으면 별도 로그인 없이 바로 메뉴를 볼 수 있기를  
**So that** 대기 시간 없이 즉시 주문을 시작할 수 있다

**인수 기준**:
- Given 태블릿에 매장/테이블 정보가 저장되어 있을 때 When 태블릿을 켜면 Then 자동으로 로그인되어 메뉴 화면이 표시된다
- Given 자동 로그인이 성공했을 때 When 메뉴 화면이 표시되면 Then 현재 테이블 번호가 화면에 표시된다
- Given 저장된 인증 정보가 만료되었을 때 When 태블릿을 켜면 Then 관리자에게 재설정이 필요하다는 안내가 표시된다

### US-02: 테이블 태블릿 초기 설정 (관리자)
**As a** 관리자 (P4)  
**I want** 태블릿에 매장ID, 테이블번호, 비밀번호를 1회 설정할 수 있기를  
**So that** 고객이 별도 로그인 없이 바로 사용할 수 있다

**인수 기준**:
- Given 초기 설정 화면에서 When 매장ID, 테이블번호, 비밀번호를 입력하고 저장하면 Then 인증 정보가 로컬에 저장되고 자동 로그인이 활성화된다
- Given 잘못된 매장ID를 입력했을 때 When 저장을 시도하면 Then 에러 메시지가 표시되고 저장되지 않는다

## Epic 2: 고객 여정 - 메뉴 탐색

### US-03: 카테고리 사이드바로 메뉴 탐색
**As a** 고객 (P1, P2, P3)  
**I want** 왼쪽 사이드바에서 카테고리를 선택하여 빠르게 이동할 수 있기를  
**So that** 원하는 메뉴를 빠르게 찾을 수 있다

**인수 기준**:
- Given 메뉴 화면이 표시될 때 When 왼쪽 사이드바를 보면 Then 모든 카테고리 목록이 표시된다
- Given 메뉴를 스크롤할 때 When 특정 카테고리 영역에 도달하면 Then 사이드바에서 해당 카테고리가 자동 하이라이트된다
- Given 사이드바에서 When 카테고리를 클릭하면 Then 해당 카테고리 영역으로 자동 스크롤된다

### US-04: 카테고리별 영역 구분된 메뉴 목록
**As a** 고객 (P2, P3)  
**I want** 전체 메뉴를 스크롤할 때 카테고리별로 구분선과 소제목이 있기를  
**So that** 현재 어떤 카테고리를 보고 있는지 명확히 알 수 있다

**인수 기준**:
- Given 전체 메뉴를 스크롤할 때 When 카테고리가 바뀌는 경계에 도달하면 Then 카테고리 소제목과 구분선이 표시된다
- Given 메뉴 카드가 표시될 때 When 각 메뉴를 보면 Then 메뉴명, 가격, 설명, 이미지가 카드 형태로 표시된다

### US-05: 인기 메뉴 확인
**As a** 고객 (P1, P2)  
**I want** 인기 메뉴에 뱃지가 표시되기를 (인기 메뉴는 우선 노출한다)
**So that** 어떤 메뉴가 인기 있는지 쉽게 알 수 있다

**인수 기준**:
- Given 메뉴 목록이 표시될 때 When 인기 메뉴 항목을 보면 Then 이미지 영역 오른쪽 상단에 인기 뱃지가 표시된다

### US-06: 메뉴 옵션 선택
**As a** 고객 (P2, P3)  
**I want** 메뉴 선택 시 맵기, 사이즈, 토핑 등 옵션을 선택할 수 있기를  
**So that** 내 취향에 맞게 커스터마이징할 수 있다

**인수 기준**:
- Given 옵션이 있는 메뉴를 선택했을 때 When 메뉴 상세 화면이 열리면 Then 옵션 그룹별 선택 UI가 표시된다
- Given 필수 옵션 그룹이 있을 때 When 필수 옵션을 선택하지 않고 장바구니 추가를 시도하면 Then 필수 옵션 선택 안내 메시지가 표시된다
- Given 옵션을 선택할 때마다 When 추가 가격이 있는 옵션을 선택하면 Then 총 가격이 실시간으로 업데이트된다
- Given 옵션이 없는 메뉴를 선택했을 때 When 메뉴를 터치하면 Then 상세 화면 없이 바로 장바구니에 추가되고 추가되었다는 툴팁 메세지가 장바구니 버튼 위에 표시된다.

## Epic 3: 고객 여정 - 장바구니 관리

### US-07: 장바구니에 메뉴 추가
**As a** 고객 (P1, P2, P3)  
**I want** 선택한 메뉴와 옵션을 장바구니에 담을 수 있기를  
**So that** 여러 메뉴를 모아서 한 번에 주문할 수 있다

**인수 기준**:
- Given 메뉴 상세에서 옵션을 선택한 후 When 장바구니 추가 버튼을 누르면 Then 메뉴+옵션이 장바구니에 추가되고 총 금액이 업데이트된다
- Given 같은 메뉴를 다른 옵션으로 추가했을 때 When 장바구니를 확인하면 Then 별도 항목으로 표시된다

### US-08: 장바구니 수정 및 삭제
**As a** 고객 (P2, P3)  
**I want** 장바구니 내 메뉴의 수량을 조절하거나 삭제할 수 있기를  
**So that** 주문 전에 자유롭게 수정할 수 있다

**인수 기준**:
- Given 장바구니에 메뉴가 있을 때 When 수량 증가/감소 버튼을 누르면 Then 수량과 총 금액이 실시간 업데이트된다
- Given 장바구니에 메뉴가 있을 때 When 삭제 버튼을 누르면 Then 해당 항목이 제거되고 총 금액이 재계산된다
- Given 장바구니에 메뉴가 있을 때 When 장바구니 비우기를 누르면 Then 모든 항목이 삭제된다

### US-09: 장바구니 유지 (새로고침)
**As a** 고객 (P1, P2, P3)  
**I want** 페이지를 새로고침해도 장바구니가 유지되기를  
**So that** 실수로 새로고침해도 주문 내용을 잃지 않는다

**인수 기준**:
- Given 장바구니에 메뉴가 담겨있을 때 When 페이지를 새로고침하면 Then 장바구니 내용이 그대로 유지된다

## Epic 4: 고객 여정 - 메뉴 추천 및 주문

### US-10: 함께 주문한 메뉴 추천
**As a** 고객 (P2, P3)  
**I want** 주문하기 직전에 다른 고객들이 함께 주문한 메뉴를 추천받기를  
**So that** 놓친 메뉴가 있는지 확인하고 추가할 수 있다

**인수 기준**:
- Given 장바구니에서 "주문하기"를 누르면 Then 바텀시트에 추천 메뉴가 이미지, 메뉴명, 가격과 함께 표시된다 (만약 추천 메뉴 정책에 걸리는 메뉴가 없다면 랜덤으로 메뉴 추천해준다)
- Given 추천 바텀시트에서 When 추천 메뉴를 선택하면 Then 해당 메뉴가 장바구니에 추가된다
- Given 추천 바텀시트에서 When "바로 주문하기" 또는 "추천 건너뛰기"를 누르면 Then 주문 확정 단계로 이동한다
- Given 장바구니에서 "주문하기"를 누른 후 When 추천 메뉴가 없으면 Then 바텀시트 없이 바로 주문 확정 단계로 이동한다

### US-11: 주문 확정
**As a** 고객 (P1, P2, P3)  
**I want** 최종 주문 내역을 확인하고 주문을 확정할 수 있기를  
**So that** 정확한 주문이 전달되었는지 확인할 수 있다

**인수 기준**:
- Given 주문 확정 화면에서 When 주문 내역을 확인하면 Then 모든 메뉴, 옵션, 수량, 금액이 정확히 표시된다
- Given 주문 확정 화면에서 When "계산 안내" 영역을 보면 Then "계산은 매장 나가기 할 때 프론트에서 해주세요" 멘트가 표시된다
- Given 주문 확정 버튼을 눌렀을 때 When 주문이 성공하면 Then 주문번호가 표시되고 장바구니가 비워지며 메뉴 화면으로 리다이렉트된다
- Given 주문 확정 버튼을 눌렀을 때 When 주문이 실패하면 Then 에러 메시지가 표시되고 장바구니는 유지된다

### US-12: 추가 주문 (n회차)
**As a** 고객 (P2, P3)  
**I want** 첫 주문 이후에도 추가 주문을 할 수 있기를  
**So that** 식사 중 필요한 메뉴를 추가로 주문할 수 있다

**인수 기준**:
- Given 첫 주문이 완료된 상태에서 When 메뉴를 다시 선택하여 주문하면 Then 해당 주문이 "n회차 추가 주문"으로 기록된다
- Given 추가 주문이 완료되면 When 주문 내역을 확인하면 Then 회차별로 구분된 뱃지가 표시된다

## Epic 5: 고객 여정 - 주문 대기 및 확인

### US-13: 주문 진행 상황 확인 (모래시계)
**As a** 고객 (P1, P2, P3)  
**I want** 주문 후 준비 진행 상황을 모래시계 애니메이션과 %로 확인할 수 있기를  
**So that** 음식이 언제 나올지 예상할 수 있다

**인수 기준**:
- Given 주문이 접수된 후 When 대기 화면을 보면 Then "주문 접수 → 준비 중 → 서빙 대기" 진행 단계가 표시된다
- Given "준비 중" 상태일 때 When 대기 화면을 보면 Then 모래시계 애니메이션과 예상 남은 시간 %가 표시된다
- Given 추가 주문이 발생했을 때 When 대기 화면의 예상 시간을 보면 Then 추가분만큼 시간이 증가하여 표시된다

### US-14: 대기 화면에서 영수증 확인
**As a** 고객 (P3)  
**I want** 대기 화면에서 현재까지의 전체 주문 영수증을 확인할 수 있기를  
**So that** 총 주문 금액과 항목을 정확히 파악할 수 있다

**인수 기준**:
- Given 대기 화면에서 When 영수증 확인 버튼을 누르면 Then 회차별 주문 항목, 옵션, 금액, 합계가 표시된다
- Given 영수증 화면에서 When 추가 주문 내역을 보면 Then "n회차 추가 주문" 뱃지로 구분되어 표시된다

### US-15: 주문 내역 조회
**As a** 고객 (P2, P3)  
**I want** 현재 세션의 전체 주문 내역을 시간순으로 확인할 수 있기를  
**So that** 무엇을 주문했는지 확인하고 총 금액을 파악할 수 있다

**인수 기준**:
- Given 주문 내역 화면에서 When 주문 목록을 보면 Then 주문번호, 시각, 메뉴+옵션, 수량, 금액, 상태, 회차 뱃지가 표시된다
- Given 주문 내역 화면에서 When 맨 하단을 보면 Then 전체 total 주문 금액이 표시된다
- Given 현재 세션의 주문만 표시되며 When 이전 세션의 주문은 Then 표시되지 않는다

## Epic 6: 관리자 여정 - 인증 및 모니터링

### US-16: 관리자 로그인
**As a** 관리자 (P4)  
**I want** 매장ID, 사용자명, 비밀번호로 관리 시스템에 로그인할 수 있기를  
**So that** 내 매장의 주문을 관리할 수 있다

**인수 기준**:
- Given 로그인 화면에서 When 올바른 매장ID, 사용자명, 비밀번호를 입력하면 Then 로그인 성공하여 대시보드로 이동한다
- Given 로그인 성공 후 When 16시간이 경과하면 Then 자동으로 로그아웃된다
- Given 로그인 화면에서 When 5회 이상 잘못된 비밀번호를 입력하면 Then 로그인 시도가 일시적으로 차단된다
- Given 로그인 상태에서 When 브라우저를 새로고침하면 Then 세션이 유지되어 재로그인이 필요없다

### US-17: 실시간 주문 대시보드
**As a** 관리자 (P4)  
**I want** 모든 테이블의 주문 현황을 그리드 형태의 대시보드로 실시간 확인할 수 있기를  
**So that** 매장 전체 주문 상황을 한눈에 파악할 수 있다

**인수 기준**:
- Given 대시보드 화면에서 When 테이블별 카드를 보면 Then 테이블번호, 총 주문액, 최신 주문 미리보기가 표시된다
- Given 새 주문이 들어왔을 때 When 대시보드를 보면 Then 2초 이내에 해당 테이블 카드가 업데이트되고 시각적으로 강조된다
- Given 대시보드에서 When 테이블별 필터를 적용하면 Then 선택한 테이블만 표시된다

### US-18: 주문 상세 확인 및 상태 변경
**As a** 관리자 (P4)  
**I want** 주문 카드를 클릭하여 상세 메뉴 목록을 보고 상태를 변경할 수 있기를  
**So that** 주문을 순서대로 처리할 수 있다

**인수 기준**:
- Given 대시보드에서 When 주문 카드를 클릭하면 Then 전체 메뉴 목록, 옵션, 수량이 상세 표시된다
- Given 주문 상세에서 When 상태 변경 버튼을 누르면 Then "대기중 → 준비중 → 완료"로 상태가 변경된다

## Epic 7: 관리자 여정 - 테이블 관리

### US-19: 주문 삭제 (직권 수정)
**As a** 관리자 (P4)  
**I want** 잘못된 주문을 삭제할 수 있기를  
**So that** 오류 주문을 정정하고 정확한 금액을 유지할 수 있다

**인수 기준**:
- Given 주문 상세에서 When 삭제 버튼을 누르면 Then 확인 팝업이 표시된다
- Given 확인 팝업에서 When 삭제를 확정하면 Then 주문이 삭제되고 테이블 총 주문액이 재계산된다
- Given 삭제가 완료되면 When 대시보드를 보면 Then 성공 피드백이 표시된다

### US-20: 테이블 이용 완료 (세션 종료)
**As a** 관리자 (P4)  
**I want** 고객 퇴장 시 테이블을 이용 완료 처리할 수 있기를  
**So that** 새 고객이 이전 주문 내역 없이 깨끗하게 시작할 수 있다

**인수 기준**:
- Given 테이블 관리에서 When "이용 완료" 버튼을 누르면 Then 확인 팝업이 표시된다
- Given 확인 팝업에서 When 이용 완료를 확정하면 Then 해당 세션의 주문이 과거 이력으로 이동되고 테이블이 리셋된다
- Given 이용 완료 후 When 새 고객이 주문을 시작하면 Then 새로운 세션이 시작되고 이전 내역은 표시되지 않는다

### US-21: 과거 주문 내역 조회
**As a** 관리자 (P4)  
**I want** 테이블별 과거 주문 내역을 조회할 수 있기를  
**So that** 매출 확인 및 운영 분석에 활용할 수 있다

**인수 기준**:
- Given 테이블 관리에서 When "과거 내역" 버튼을 누르면 Then 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다
- Given 과거 내역 화면에서 When 날짜 필터를 적용하면 Then 해당 기간의 주문만 표시된다
- Given 과거 내역 화면에서 When "닫기" 버튼을 누르면 Then 대시보드로 복귀한다

## Epic 8: 관리자 여정 - 메뉴 및 옵션 관리

### US-22: 메뉴 CRUD
**As a** 관리자 (P4)  
**I want** 메뉴를 등록, 수정, 삭제하고 노출 순서를 조정할 수 있기를  
**So that** 메뉴판을 최신 상태로 유지할 수 있다

**인수 기준**:
- Given 메뉴 관리 화면에서 When 새 메뉴를 등록하면 Then 메뉴명, 가격, 설명, 카테고리, 이미지가 저장된다
- Given 메뉴 관리에서 When 필수 필드를 비워두고 저장하면 Then 검증 에러 메시지가 표시된다
- Given 메뉴 관리에서 When 메뉴를 삭제하면 Then 확인 후 메뉴가 제거된다
- Given 메뉴 관리에서 When 메뉴 노출 순서를 변경하면 Then 고객 화면에 변경된 순서로 표시된다

### US-23: 옵션 그룹 관리
**As a** 관리자 (P4)  
**I want** 옵션 그룹을 생성하고 메뉴에 연결할 수 있기를  
**So that** 고객이 메뉴를 커스터마이징할 수 있다

**인수 기준**:
- Given 옵션 관리에서 When 새 옵션 그룹을 생성하면 Then 그룹명, 필수/선택 구분이 설정된다
- Given 옵션 그룹에 When 옵션 항목을 추가하면 Then 항목명과 추가 가격이 설정된다
- Given 옵션 그룹을 When 메뉴에 연결하면 Then 해당 메뉴의 상세 화면에서 옵션이 표시된다
- Given 이미 생성된 옵션 그룹을 When 다른 메뉴에 연결하면 Then 동일한 옵션 그룹이 재사용된다

## Epic 9: 에러 및 예외 시나리오

### US-24: 네트워크 오류 처리 (고객)
**As a** 고객 (P1, P2, P3)  
**I want** 네트워크 오류 시 명확한 안내를 받기를  
**So that** 상황을 이해하고 적절히 대응할 수 있다

**인수 기준**:
- Given 주문 전송 중 When 네트워크 오류가 발생하면 Then "네트워크 연결을 확인해 주세요" 에러 메시지가 표시되고 장바구니가 유지된다
- Given 메뉴 조회 중 When 서버 응답이 없으면 Then 재시도 버튼이 포함된 에러 화면이 표시된다

### US-25: 세션 만료 처리 (고객)
**As a** 고객 (P1, P2, P3)  
**I want** 세션이 만료되면 자동으로 재인증되기를  
**So that** 주문 과정이 중단되지 않는다

**인수 기준**:
- Given 태블릿 세션이 만료되었을 때 When API 요청이 발생하면 Then 저장된 인증 정보로 자동 재인증을 시도한다
- Given 자동 재인증이 실패했을 때 When 화면을 보면 Then 관리자 호출 안내 메시지가 표시된다

### US-26: 폴링 실패 처리 (관리자)
**As a** 관리자 (P4)  
**I want** 폴링 연결이 끊어지면 알림을 받기를  
**So that** 주문 누락 없이 대응할 수 있다

**인수 기준**:
- Given 주문 폴링이 실패했을 때 When 대시보드를 보면 Then "실시간 업데이트 중단" 경고가 표시된다
- Given 폴링이 복구되면 When 대시보드를 보면 Then 누락된 주문이 포함된 최신 데이터로 갱신된다
