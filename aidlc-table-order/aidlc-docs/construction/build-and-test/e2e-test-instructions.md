# E2E 테스트 지침

## 도구
- **Playwright** (권장) 또는 Cypress
- 전체 스택 (DB + backend + customer + admin) 실행 상태에서 수행

## 설치
```bash
cd aidlc-table-order
npm install -D @playwright/test
npx playwright install
```

## E2E 테스트 시나리오

### E2E-01: 고객 주문 전체 플로우
```
1. 고객앱 접속 (/menu)
2. 카테고리 사이드바에서 "메인 메뉴" 클릭
3. "김치찌개" 메뉴 카드 클릭
4. 맵기 옵션 "보통맛" 선택
5. 장바구니 추가 버튼 클릭
6. 장바구니 페이지로 이동
7. 수량 2로 변경
8. "주문하기" 클릭
9. 추천 바텀시트 "바로 주문하기" 클릭
10. 주문 확정 화면에서 "계산 안내 멘트" 확인
11. "주문 확정" 클릭
12. 주문번호 표시 확인
13. 대기 화면 진행률 확인
14. 영수증 확인 버튼 클릭 → 회차 영수증 확인
```

### E2E-02: 추가 주문 (n회차)
```
1. E2E-01 완료 후
2. 메뉴로 돌아가 "콜라" 선택 (옵션 없는 메뉴)
3. 툴팁 "장바구니에 추가됨" 확인
4. 장바구니 → 주문하기 → 주문 확정
5. 주문 내역에서 "2회차 추가 주문" 뱃지 확인
6. 영수증에서 1회차, 2회차 구분 확인
7. total 금액 확인
```

### E2E-03: 관리자 주문 관리
```
1. 관리자앱 접속 → 로그인 (admin / password123)
2. 대시보드에서 테이블 카드 확인
3. 주문 카드 클릭 → 상세 모달
4. "준비중" 버튼 클릭 → 상태 변경 확인
5. "완료" 버튼 클릭 → 상태 변경 확인
6. 고객앱 대기 화면에서 상태 반영 확인
```

### E2E-04: 관리자 테이블 이용 완료
```
1. 고객이 주문 완료한 상태
2. 관리자앱 → 테이블 카드 → "이용 완료" 클릭
3. 확인 팝업 → 확정
4. 테이블 카드 리셋 확인 (총 금액 0)
5. "과거 내역" 클릭 → 아카이빙된 주문 확인
6. 고객앱에서 주문 내역 빈 화면 확인
```

### E2E-05: 관리자 메뉴/옵션 관리
```
1. 관리자앱 → 메뉴 관리
2. "새 메뉴 추가" → 폼 작성 → 저장
3. 고객앱에서 새 메뉴 표시 확인
4. 옵션 관리 → 새 옵션 그룹 생성
5. 메뉴에 옵션 연결
6. 고객앱에서 옵션 선택 가능 확인
```

### E2E-06: 에러 처리
```
1. 백엔드 중지 상태에서 고객앱 접속
2. 에러 배너 표시 확인
3. 재시도 버튼 동작 확인
4. 관리자앱 → 폴링 실패 경고 표시 확인
```

## 실행
```bash
npx playwright test
npx playwright test --headed  # 브라우저 보기 모드
npx playwright show-report    # 결과 리포트
```
