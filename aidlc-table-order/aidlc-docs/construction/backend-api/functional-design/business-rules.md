# 비즈니스 규칙 - backend-api

## BR-01: 인증 규칙

### BR-01-1: 비밀번호 정책
- 최소 8자 이상
- bcrypt 해싱 (adaptive algorithm)
- 평문 비밀번호는 절대 저장/로깅하지 않음

### BR-01-2: 로그인 시도 제한
- 5회 연속 실패 시 계정 15분 잠금
- 잠금 중 로그인 시도 시 "잠김" 에러 반환 (남은 시간 포함)
- 로그인 성공 시 시도 횟수 즉시 리셋

### BR-01-3: JWT 세션
- 만료 시간: 16시간
- 페이로드: role (admin/table), store_id, subject (admin_id 또는 table_id)
- 서버 측 서명 검증 필수 (매 요청)
- Refresh token으로 갱신 가능

### BR-01-4: 역할 기반 접근 제어
- admin: 관리자 전용 엔드포인트 (주문 상태 변경, 삭제, 테이블 관리, 메뉴 CRUD)
- table: 고객 전용 엔드포인트 (메뉴 조회, 주문 생성, 주문 내역 조회)
- 자기 매장 데이터만 접근 가능 (store_id 검증)

## BR-02: 메뉴/옵션 규칙

### BR-02-1: 메뉴 필수 필드
- 메뉴명: 1~100자, NOT NULL
- 가격: 0 이상 정수 (원 단위)
- 카테고리: 반드시 하나 지정

### BR-02-2: 옵션 그룹 규칙
- 옵션 그룹명: 1~50자
- 옵션 항목 추가 가격: 0 이상 정수
- 필수 그룹(is_required=true): 주문 시 반드시 1개 이상 선택
- 선택 그룹(is_required=false): 선택 안 해도 됨
- 하나의 옵션 그룹을 여러 메뉴에 연결 가능 (N:M)
- 하나의 메뉴에 여러 옵션 그룹 연결 가능

### BR-02-3: 메뉴 삭제 규칙
- 메뉴 삭제 시 연결된 MenuOptionGroup 자동 해제
- 이미 주문된 메뉴는 OrderItem에 스냅샷이 있으므로 안전하게 삭제 가능
- 옵션 그룹 삭제 시 연결된 MenuOptionGroup 자동 해제

### BR-02-4: 인기 메뉴
- is_popular=true: 관리자 수동 지정 (항상 인기)
- 자동 판별: 최근 30일 주문 횟수 기준 매장 상위 20%
- 인기 메뉴는 카테고리 내에서 우선 노출 (sort 우선순위: 인기 > sort_order)

## BR-03: 주문 규칙

### BR-03-1: 주문 생성 검증
- 최소 1개 이상의 주문 항목 필요
- 각 메뉴는 is_available=true여야 함
- 수량은 1 이상 정수
- 필수 옵션 그룹에서 반드시 1개 이상 선택
- 옵션은 해당 메뉴에 연결된 그룹의 항목만 허용

### BR-03-2: 금액 계산
- 항목 소계 = (메뉴 단가 + 옵션 추가 가격 합계) × 수량
- 주문 총액 = 모든 항목 소계의 합
- 서버 측 금액 계산만 유효 (클라이언트 전송 금액은 무시)

### BR-03-3: 주문 회차
- 세션 내 첫 주문: round = 1
- 이후 추가 주문: round = 이전 최대 round + 1
- 주문 삭제해도 회차 번호는 재사용하지 않음

### BR-03-4: 주문 번호
- 형식: YYYYMMDD-NNNN (예: 20260408-0001)
- 당일 매장별 순번 자동 증가
- 매일 0001부터 시작

### BR-03-5: 주문 상태 전이
- pending → preparing (관리자만)
- preparing → completed (관리자만)
- 역방향 전이 금지
- 건너뛰기 금지 (pending → completed 불가)
- completed 상태에서 추가 변경 불가

### BR-03-6: 주문 삭제
- 관리자만 삭제 가능
- 모든 상태의 주문 삭제 가능
- 삭제 시 해당 세션의 총 주문액 재계산
- 삭제된 주문 번호는 재사용하지 않음

## BR-04: 세션 규칙

### BR-04-1: 세션 시작
- 첫 주문 생성 시 활성 세션이 없으면 자동 생성
- 하나의 테이블에 동시에 하나의 활성 세션만 존재

### BR-04-2: 이용 완료 (세션 종료)
- 관리자만 실행 가능
- 처리 순서: 주문 아카이빙 → 원본 삭제 → 세션 완료
- 아카이빙: Order + OrderItem + OrderItemOption → OrderHistory (items_snapshot JSON)
- 완료 후 테이블의 현재 주문 목록 = 빈 상태
- 완료 후 새 주문 시 새 세션 자동 생성

### BR-04-3: 과거 내역 조회
- OrderHistory에서 table_id + 날짜 범위로 필터링
- 시간 역순 정렬
- 세션별 그룹핑 가능 (session_id)

---

## BR-05: 추천 규칙

### BR-05-1: 추천 메뉴 생성
- 장바구니 메뉴와 함께 주문된 적 있는 메뉴 중 장바구니에 없는 것
- 출현 빈도순 상위 5개
- is_available=true인 메뉴만 추천

### BR-05-2: 랜덤 폴백
- 히스토리 기반 추천 결과가 0개인 경우
- 해당 매장의 is_available=true 메뉴 중 장바구니에 없는 것을 랜덤 5개

---

## BR-06: 대기 시간 규칙

### BR-06-1: 예상 시간 계산
- 기본: 최근 7일 동일 메뉴 평균 준비 시간
- 폴백: 데이터 없으면 매장 default_prep_time_minutes
- 병렬 조리 가정: 주문 내 최대 시간 적용
- 앞선 주문 대기: 앞 주문 수 × 평균 50% 추가

### BR-06-2: 진행률 표시
- 진행률 = (경과 시간 / 총 예상 시간) × 100
- 최솟값: 0%, 최댓값: 99% (100%는 completed 상태에서만)
- 추가 주문 시 예상 시간 재계산
